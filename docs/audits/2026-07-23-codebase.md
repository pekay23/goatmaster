# Codebase Audit — 2026-07-23

## Current state

The project has undergone a significant refactoring and feature expansion phase since the v2.0 audit. Key milestones achieved:

- **Component extraction:** 22 files extracted from monolithic panels (EventsPanel, HealthPanel, SalesPanel, BreedingPanel, MainApp, SmartScanner)
- **Navigation consolidation:** 14 individual bottom tabs → 4 groups (Herd, Ops, Intel, Settings) with sub-tab pills
- **Admin portal:** Dashboard, Users CRUD, Tiers management with full CSS v2.0
- **Admin API:** 7 REST endpoints for stats, users, tiers
- **ML service migration:** Render.com → Fly.io (always-on, 1GB RAM, no cold starts)
- **Login UX fixes:** Password toggle, email field correction, response unwrapping fix
- **Database:** `tiers` table created with Free/Basic/Pro seed data

## Known issues

### 1. `is_active` column doesn't exist

- **File:** `app/api/admin/users/[id]/route.js`
- **Impact:** The Active toggle in the admin users panel is visual-only. The API returns `is_active: true` as a virtual field but never persists it.
- **Fix needed:** Add `is_active BOOLEAN DEFAULT true` column to the `users` table in `scripts/migrate.js`.

### 2. `goat_embeddings` table doesn't exist

- **File:** `app/api/admin/stats/route.js`
- **Impact:** Admin dashboard shows 0 for total scans and total embeddings. The API catches the error gracefully but the data is missing.
- **Fix needed:** Create the `goat_embeddings` table with the pgvector extension. Schema includes `id UUID PK`, `goat_id UUID FK`, `user_id INTEGER`, `embedding VECTOR(1024)`, `image_url TEXT`.

### 3. Admin auth uses localStorage

- **File:** `app/admin/layout.jsx`
- **Impact:** Admin authentication reads `goat_user` from localStorage, not the JWT cookie. This is XSS-vulnerable and doesn't respect JWT expiry. A user with `document.cookie` access can't be impersonated, but JS running on the page can read localStorage and impersonate an admin.
- **Fix needed:** Migrate admin auth to use the same JWT cookie pattern as the rest of the app. `requireAuth()` should check `role === 'admin'` before allowing admin route access.

### 4. Scan count hardcoded to 0

- **File:** `app/api/admin/stats/route.js`
- **Code:** `totalScans: 0`
- **Impact:** The scans stat card always shows 0, even if scans have been performed.
- **Fix needed:** Query a scans table or count from the ML service logs when available.

### 5. Hydration mismatch warnings (benign)

- **Files:** `app/layout.jsx`
- **Cause:** Browser extensions (anti-fingerprinting tools like CanvasBlocker) inject `__processed_*` attributes into the DOM after server rendering.
- **Mitigation:** `suppressHydrationWarning` on `<html>` and `<body>` tags. Warnings are harmless and don't affect functionality.

### 6. Database migration requires env vars

- **Impact:** Running `scripts/migrate.js` locally fails with `ECONNREFUSED` if `DATABASE_URL` isn't set in the environment. The script doesn't load `.env.local` automatically.
- **Workaround:** `bun --env-file=.env.local run scripts/migrate.js`

### 7. No audit logging for admin actions

- **Impact:** Admin user creation, updates, and deletions are not logged. There's no record of who changed what.
- **Fix needed:** Add an `audit_logs` table and log all admin actions with timestamp, admin user ID, action type, and target user ID.

## Recommendations

1. **Short-term:** Add `is_active` column and `goat_embeddings` table to migration script. These are quick wins.
2. **Medium-term:** Migrate admin auth to JWT. This is a security improvement but requires updating all admin API routes and the admin layout.
3. **Medium-term:** Add audit logging for admin actions. Essential for accountability in a multi-admin scenario.
4. **Long-term:** Add billing integration (Stripe) so tier changes trigger actual payments.
