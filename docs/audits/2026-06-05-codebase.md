# 2026-06-05 · Codebase audit (v2.0 prep)

A full-repo review done before the v2.0 upgrade (bun migration, color system rewrite, doc refresh).

## Scope

- `app/**`, `components/**`, `lib/**`, `ml_service/**`, `package.json`, `app/globals.css`

## Findings

### 1. Package manager — npm → bun

- **Before:** both `bun.lock` and `package-lock.json` committed; `dev` script used `NODE_OPTIONS='--max_old_space_size=3072' next dev --turbo --webpack` (also `--webpack` was redundant with `--turbo`).
- **After:** packageManager pinned to `bun@1.2.0`; engines pinned to `node ≥ 20` + `bun ≥ 1`; dev script reduced to `next dev --turbo` (no 3 GB heap ceiling, no `--webpack`).
- **Risk:** none — the code never depended on Node-specific behaviour.

### 2. Color system — primitive + semantic

- **Before:** ~10 hand-picked CSS variables (`--bg-app`, `--primary`, etc.) with light + dark, but no primitive scale, no documented tokens, no WCAG verification.
- **After:** full two-layer system in `app/globals.css`:
  - Primitives: `--pasture-*`, `--sun-*`, `--barn-*`, `--sky-*`, `--slate-*` (Tailwind-style scales, 10/6/6/2/12 stops)
  - Semantic: `--bg-app`, `--text-main`, `--primary`, `--danger`, `--focus-ring`, etc.
  - All semantic tokens documented in `docs/design/02-colors.md` with light + dark HSL + hex.
  - All text values clear WCAG AA (4.5:1) on their intended surface.
- **Migration note:** every component was already using `var(--bg-…)`, `var(--primary)`, `var(--text-…)` — so the rewrite was a one-file change to `globals.css`.

### 3. Documentation

- **Before:** a single `docs/LLM_COUNCIL_FRAMEWORK.md` and a thin top-level `README.md`.
- **After:** full template (matching `aerojet-academy/docs` and `raymond-gray-platform/docs`):
  - `architecture/` — system overview, data flow, database, ML architecture, security model, API ref, deployment
  - `design/` — foundations, colors, typography, components, motion
  - `guides/` — setup, development, deployment, handover, contributing
  - `audits/` — this file + known-issues
  - `plans/` — future plans, v2-upgrade, ml-retraining-loop
  - `CHANGELOG.md` and `README.md` index

### 4. Security

- All API routes still gate on JWT cookie (no change).
- IndexedDB cache still cleared on logout and on account delete (no change).
- `MIGRATE_SECRET` gate kept; consider rotating on prod (out of scope here).

### 5. Performance

- No regressions. The dev-server memory cap is removed because Turbopack is now the default and doesn't need the 3 GB ceiling.
- Production build is unchanged.
